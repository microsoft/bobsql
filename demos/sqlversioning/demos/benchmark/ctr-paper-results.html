Throughput Degradation (CTR vs TempDB Versioning)

WorkloadThroughput Degradation
TPC-C (update intensive)13.8%
TPC-E (mixed read/write)2.4%

Measured as % throughput reduction vs traditional versioning in TempDB. 1% of transactions explicitly rolled back. TPC-C is worst-case (extremely update intensive); TPC-E represents typical OLTP.

Storage Impact (50M-row single transaction, 200-byte rows)

MetricTraditional RecoveryWith ADR
Transaction LogUp to 38 GB (grows with txn size)Capped at ~200 MB (continuous truncation)
Database Growth (PVS)None908 MB / ~9% growth (updates only, diff-based)

ADR truncates the log continuously (~200 MB cap). Trade-off: versions for updates are stored in a Persistent Version Store (PVS) inside the user database. Inserts/deletes use in-row flags — no PVS growth.
