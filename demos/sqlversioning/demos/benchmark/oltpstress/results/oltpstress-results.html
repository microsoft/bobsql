⚠ OL Result is a Workload Artifact, Not OL Overhead

OL eliminates lock escalation on batch updates, so those 10 threads run 5× more batches (2,414 vs 311–491). Each batch touches ~2,500 rows, so OL generates ~5.8M row modifications vs ~700K–1.2M in other configs. This floods the log writer and PVS, starving point-update threads (7.36ms vs 1.54–1.98ms). The per-row cost of OL is zero—the system is simply doing 5× more work.
